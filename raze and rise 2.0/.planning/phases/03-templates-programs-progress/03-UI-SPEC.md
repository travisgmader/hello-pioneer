# Phase 3 — UI Design Contract

> Visual and interaction contract for Templates, Programs & Progress (template management, exercise library, program builder, progress charts, measurement history, progress photos, gamification).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Inherits and extends Phase 2 (which extends Phase 1).** All Phase 1 and Phase 2 tokens (colors, spacing, typography, motion, radius) are unchanged. Phase 3 introduces NEW tokens for charts, segment tabs, badge states, and progress photo UI. The Phase 1/2 "Phase-to-Phase Token Stability Contract" is honored — no Phase 1 or Phase 2 token is redefined.
>
> Aesthetic target (DESIGN-02): **Whoop / Strong** for data-dense screens (charts, session history). Gothic serious-athlete brand moments (badge unlocks). Gold-on-obsidian. No lifestyle softness, no celebration confetti.

## Spacing Scale

Phase 3 uses the Phase 1/2 8-point scale unchanged. No new spacing tokens introduced.

| Token | Value | Phase 3 Usage |
|-------|-------|---------------|
| `xs` | 4px | Chip-to-chip gap in muscle group filter row, stat icon-to-label gap, badge grid cell padding, segment tab indicator thickness |
| `sm` | 8px | Template card internal vertical gap, exercise result row gap, stats row number-to-label gap, date range chip gap |
| `md` | 16px | Screen edge horizontal padding (carry), template card internal padding, segment tab horizontal padding per tab, chart area horizontal margin |
| `lg` | 24px | Section padding between segment content areas, gap between stats row and sessions list, chart section top padding |
| `xl` | 32px | Top spacing from screen header to first content section, gap between chart area and measurement chart |
| `2xl` | 48px | Empty state vertical padding (no templates, no chart data, no photos) |
| `3xl` | 64px | (unused in Phase 3 primary flows — reserved for modal empty states) |

**Exceptions (Phase 3 — declared):**

- **Stats row stat block: 64pt minimum height** — each stat (number + label stacked) must be touchable and readable at arm's length; 64pt gives the number room to breathe at 28px NumericDisplay size
- **SegmentTab touch target: 44pt height** — segment tabs are inside the screen, not the tab bar; must meet 44pt minimum per iOS HIG
- **BadgeCard: 72×72pt** — badge icon is 48×48pt visually, 72×72pt tap target (grid layout of 3 per row; 72pt fits 3 columns on 375pt-wide screens with `md` margins and `sm` gaps)
- **TemplateCard: 64pt minimum height** — template name + day label + exercise count; compact but above 44pt
- **ExerciseResultRow: 56pt height** — consistent with Phase 2 ExerciseCard row density (exercise name + muscle group category)
- **SessionHistoryRow: 56pt height** — date + day label + go-rate in one scan line; matches Phase 2 ExerciseCard row for consistency
- **Chart area: 200pt minimum height** — Victory Native XL needs adequate vertical space to render axis labels legibly; 200pt is the floor, 240pt preferred
- **Progress photo thumbnail: 80×80pt** — chronological gallery thumbnails; three per row with `xs` gap on a 375pt screen with `md` edge margins
- **Before/after comparison image: half screen width minus md margin** — fills available width evenly for side-by-side layout
- **ProgramWeekGrid cell: 44pt height** — each week row in the AI program review grid must be tappable for edit mode
- **DeloadBanner height: 56pt** — enough for heading + body on one visible line each, with icon

All exceptions are declared. None are arbitrary.

## Motion / Transitions

Carry from Phase 1/2. Phase 3 adds THREE new motion tokens.

| Token | Duration | Easing | Phase 3 Usage |
|-------|----------|--------|---------------|
| `instant` (carry) | 0ms | — | History edit field value change, segment content swap on tab switch (content is instant; only the indicator animates) |
| `fast` (carry) | 150ms | ease-out | Segment tab indicator slide, Chip selected state color change (date range, muscle group filters), TemplateCard press feedback |
| `default` (carry) | 200ms | ease-out | SegmentTabs indicator slide from one tab to another, today-card tint fade on mount, DeloadBanner slide-in from top |
| `slow` (carry) | 300ms | ease-in-out | BadgeUnlockToast enter (slide up from bottom) and exit (fade + slide down) — matches Phase 2 RestTimerPill enter/exit for consistent floating overlay language |
| **`chart-animate` (NEW)** | 400ms | ease-out | Victory Native XL chart line draw-in animation on first mount and when exercise picker selection changes. 400ms gives the line enough time to "draw" from left to right without feeling slow. |
| **`toast-hold` (NEW)** | 2500ms | — | BadgeUnlockToast hold duration before auto-dismiss begins. Not an animation duration — this is the display dwell time. After `toast-hold`, the `slow` (300ms) exit runs. Total visible: ~2.8 seconds. |
| **`segment-slide` (NEW)** | 200ms | ease-out | SegmentTab active indicator sliding horizontally. Same duration as `default`, declared separately for semantic clarity. Executor should use `withSpring` (stiffness 300, damping 30) via Reanimated for the indicator position — spring gives it a slightly physical feel over pure ease-out. |

**BadgeUnlockToast sequence (locked — D-13):**

1. `AnubisOverlay` Lottie completes → `router.replace('/(tabs)/')` fires
2. Dashboard mounts → if `newBadge` is in the navigation params, `BadgeUnlockToast` mounts
3. Toast slides UP from 48pt below its final position over `slow` (300ms) — same Reanimated pattern as RestTimerPill
4. Toast holds for `toast-hold` (2500ms)
5. Toast fades + slides down over `slow` (300ms), then unmounts
6. One badge toast max per session. If multiple badges unlocked (rare), show one toast sequentially (queue, not stack)

**Segment tab switch sequence:**

1. User taps a segment label
2. `segment-slide` spring animation slides the 2px accent bottom indicator to the new tab position
3. Active tab label transitions color: `fg-muted` → `accent` over `fast` (150ms)
4. Content area instantly swaps (no scroll animation between segments — content is separate per tab, not a scrollable carousel)
5. Light haptic: `Haptics.selectionAsync()`

**`useReducedMotion()` policy (carry):**

- When ON: BadgeUnlockToast appears instantly, holds 2500ms, disappears instantly
- When ON: SegmentTab indicator jumps to position instantly (no spring)
- When ON: Victory Native XL chart draw-in is skipped (pass `animate={false}` to VictoryChart when `isReducedMotionEnabled` is true)
- When ON: DeloadBanner appears instantly (no slide-in)

## Interaction Patterns

### Exercise Search Flow (Template Builder)

1. User opens Template Builder (new or edit mode)
2. `ExerciseSearchBar` is visible but unfocused — template exercise list (`FlashList`) visible below it
3. User taps `ExerciseSearchBar` → keyboard opens → search bar gains `border-strong` focus state
4. Template exercise list **instantly replaces** with exercise results `FlashList` — no animation on swap
5. As user types, results filter in real-time (debounced 200ms against PowerSync local exercise table)
6. User taps `MuscleGroupFilterRow` chip to narrow results — "All" chip deselects, tapped chip activates (`accent-dim` bg + `border-strong` border)
7. Tapping an `ExerciseResultRow` adds it to the template; the `Plus` icon → `Check` icon with no animation (instant); row stays visible in results (user can add more)
8. User taps `X` in `ExerciseSearchBar` OR presses keyboard dismiss → search clears → template exercise list returns
9. Newly added exercises appear at the bottom of the template exercise list
10. Haptic: `Haptics.selectionAsync()` on chip tap; `Haptics.impactAsync(Light)` on exercise add

### Set/Rep/Rest Config in Template Builder

The template builder reuses Phase 2 `SetRow` and `ExpandedSetForm` with one adaptation:

- **WeightInput is replaced** by two inputs side-by-side: "Reps low" + "Reps high" (rep range, e.g., "3" and "5" for 3–5 reps). Both use the `WeightInput` component with different placeholder text.
- Rest override: A "Rest" input in the `ExpandedSetForm` using `WeightInput` with placeholder "90s" — seconds-only integer input.
- Exercise type toggle: a 3-option row ("Standard" / "Bodyweight" / "Run") using Phase 1 `Toggle` extended to 3 options (or 3 `Chip` components with single-select behavior).
- All other Phase 2 `SetRow` behaviors (chevron expand, warm-up toggle, RPE — all stripped from template builder as they are session-time actions, not template config).

### Superset Pairing in Template Builder

1. User long-presses an `ExerciseCard` in the template list → context menu appears: "Pair as superset" + "Delete exercise"
2. "Pair as superset" → highlight mode: a "Choose partner" caption appears (Caption `fg-muted`); tapping another `ExerciseCard` pairs them
3. The two cards merge into a `SupersetPair` (Phase 2 reused) with the vertical bracket connector
4. Unpair: long-press either card in the superset → "Unpair" option in context menu
5. Haptic: `Haptics.impactAsync(Medium)` on long-press to open context menu

### Segment Tab Switching

1. User taps a segment tab label
2. `segment-slide` spring animation (200ms, stiffness 300 damping 30) slides the `SegmentIndicator` to the new tab
3. Active tab label: `fg-muted` → `accent` color transition over `fast` (150ms)
4. Content area: instant swap (no cross-fade, no swipe animation between segments)
5. `Haptics.selectionAsync()` on each tab tap
6. Scroll position per segment is preserved independently (each segment maintains its own scroll offset)

### History Edit Workflow

1. User taps a `SessionHistoryRow` in Overview → full-screen `HistoryEditView` pushes in
2. Edit view shows the session as a Phase 2-style exercise list — all fields editable
3. User edits a weight value: taps `WeightInput`, types new value, keyboard dismisses on blur
4. User changes go/no-go: taps `SetResultButton` — toggles state immediately (no confirmation)
5. User taps "Save" button in header → `writeTransaction()` commits all changes to PowerSync → PowerSync syncs to Supabase → charts re-query and reflect new data on return to Charts segment (HISTORY-03)
6. No partial save: all edits commit together on "Save". Discard path: back button shows `Alert.alert` — "Discard changes? Your edits will be lost." / "Discard" (destructive) / "Keep editing" (cancel)
7. Delete session: overflow menu → `Alert.alert` — "Delete this session? This can't be undone." / "Delete" (destructive) / "Cancel"

### Progress Photo Upload + Comparison Selection

**Upload:**
1. User taps "Upload photo" button in Photos segment
2. `expo-image-picker` `launchImageLibraryAsync` or `launchCameraAsync` opens (show action sheet to choose "Take photo" vs "Choose from library")
3. On selection: compress to max 1200px on longer side (JPEG quality 0.8) before upload
4. Upload to Supabase Storage (`progress-photos/{userId}/{iso-date}.jpg`) via `supabase.storage.from('progress-photos').upload()`
5. Show inline "Uploading..." caption below button + `Spinner`; button disabled during upload
6. On complete: gallery refreshes, new thumbnail appears at front, "Uploading..." clears
7. On error: HelperText error below button ("Upload failed. Try again.")

**Comparison:**
1. User long-presses any thumbnail → action sheet: "Select for comparison" / "Delete photo" / "Cancel"
2. Tapping "Select for comparison": gallery enters selection mode — tapped thumbnail shows `Check` overlay, a bottom bar appears: "Now select a second photo"
3. User taps a second thumbnail → `ProgressPhotoComparison` modal presents full-screen
4. Modal: left = earlier date labeled "Before", right = later date labeled "After"
5. Dismiss: Lucide `X` tap or swipe-down gesture

### AI Program Generation Flow

1. User taps "Generate with AI" in Split tab Programs section
2. `AIGenerationSheet` bottom sheet slides up (Phase 1 `slow` 300ms)
3. User fills 3 fields (goal, current lifts, duration). All are optional — Claude has defaults
4. Taps "Generate" button → button replaced by `Spinner` + "Claude is building your program..." caption
5. Edge Function called (`/functions/v1/generate-program`); on response: `AIGenerationSheet` auto-dismisses, navigation pushes `AIProgramReviewScreen`
6. On error: HelperText error in sheet "Program generation failed. Try again." — Retry is the Generate button re-tapping
7. On `AIProgramReviewScreen`: user reviews `ProgramWeekGrid`
   - "Accept program" → `writeTransaction()` saves to PowerSync → program becomes active in Split tab → navigate back
   - "Regenerate" → re-shows `AIGenerationSheet` pre-filled with previous inputs → same loading flow
   - "Edit manually" → navigate to manual program builder screen pre-filled

### Badge Unlock Toast Timing

Per D-13 — exact sequence:

1. Session completes → `AnubisOverlay` fades in → Lottie plays (~2–3 sec) → fades out
2. `router.replace('/(tabs)/')` fires → Dashboard mounts
3. **Milestone check**: on Dashboard mount, check if `newBadge` in navigation params (populated by `completeSession` transaction in Phase 2 wave 5 plan if milestone was hit)
4. If `newBadge` exists: `BadgeUnlockToast` mounts at the bottom of the Dashboard view (above bottom safe area, same position layer as `RestTimerPill`)
5. `BadgeUnlockToast` slides up over `slow` (300ms)
6. Holds for `toast-hold` (2500ms)
7. Fades + slides down over `slow` (300ms), unmounts
8. `newBadge` param cleared from navigation state
9. Achievement segment in Progress tab will show the badge as unlocked on next visit

### Deload Suggestion Trigger + Toggle

**Auto-detection trigger:** After completing a session, the `completeSession` transaction checks: has `split_settings.current_phase_weeks >= N` (default N=4)? If yes, write `deload_suggested = true` to local MMKV (not PowerSync — this is a UI hint, not a training record). On next Workouts tab mount: if `deload_suggested = true` AND `deload_acknowledged = false` → `DeloadBanner` appears.

**Manual toggle:** Split tab has a Toggle "Manual deload active" (uses Phase 1 `Toggle` binary). When activated: writes `deload_active = true` to MMKV + PowerSync `split_settings`. Workouts tab shows `DeloadBanner` in `manual` variant permanently until toggled off.

**Deload mode effect:** When `deload_active = true`: the `TodayCard` exercise count caption shows "(Deload week)" suffix. No separate screen or workout template — the deload flag is passed to the session and reduces suggested weights shown in the `ProgressiveOverloadHint` component (shows 65% of normal suggestion).

**Dismissal:** User taps `X` on `DeloadBanner` → sets `deload_acknowledged = true` in MMKV for the current week. Banner re-appears next week if still auto-detecting.

### Charts Accessibility (Victory Native XL)

Victory Native XL supports `accessibilityLabel` on the top-level `VictoryChart` wrapper. Executor MUST pass a meaningful label (see table above). For VoiceOver users, the label is the only chart information — the visual render is not traversable. For the data table accessibility alternative (Phase 6 full a11y audit), a "View as table" hidden button adjacent to each chart is a Phase 6 enhancement — not Phase 3 scope.

### Color-Independent State Communication

All Phase 3 states communicate via shape/text in addition to color:

- Active segment tab: color (`accent`) + weight (700 vs 400) + 2px indicator bar + `accessibilityState.selected`
- Locked badge: icon glyph change (Lock overlay icon) + fill color + `opacity-50` + different border
- Chart line differentiation: gold line (`chart-line`) for workout data vs. blue line (`chart-measurement`) for body measurements — both are also labeled in the chart heading; screen reader announces labels via `accessibilityLabel`
- DeloadBanner: `AlertTriangle` icon + heading text — icon is decorative reinforcement, not the only signal
- TodayBadge: pill contains text "Today" — not just color

### Dynamic Type

`allowFontScaling={false}` continues from Phase 1/2. Victory Native XL chart labels also use fixed sizes (Victory's `fontSize` style prop, not NativeWind). Dynamic Type compliance lands in Phase 6.

### Reduce Motion

- Victory Native XL chart draw-in: pass `animate={false}` when `isReducedMotionEnabled` is true
- `BadgeUnlockToast` enter/exit: instant appearance and disappearance (skip `slow` 300ms)
- `SegmentIndicator` spring: instant jump to new position
- `DeloadBanner` slide-in: instant appearance
