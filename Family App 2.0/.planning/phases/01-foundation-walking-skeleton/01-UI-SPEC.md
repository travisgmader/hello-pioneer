# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation & Walking Skeleton. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase visible surface:** Google OAuth login screen → Access Denied screen (if allowlist fails) → Family Creation Wizard (new users without a linked family) → Authenticated app shell with sticky top header + mobile bottom-tab nav + placeholder routes (Dashboard, Chores, Calendar, Meals, Groceries, Notes) → Theme toggle (Lavender / Midnight) → Offline banner + Error Boundary fallback.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage in Phase 1 |
|-------|-------|------------------|
| xs | 4px | Icon-to-text gap inside nav tabs, divider thickness, focus ring width |
| sm | 8px | Compact spacing inside the Google-button (icon ↔ label gap), error pill padding-y, between subtitle and CTA on Login |
| md | 16px | Default body padding, gap between offline banner items, gap between form rows in Family Creation Wizard |
| lg | 24px | Login card outer padding-x, page outer padding on Dashboard/placeholders, vertical rhythm between sections |
| xl | 32px | Login card vertical padding offset (top vs bottom hero spacing), Wizard form section gaps |
| 2xl | 48px | Login card max-width inner padding, vertical breathing room above Access Denied headline |
| 3xl | 64px | Mobile bottom-tab bar height baseline (before safe-area inset) |

**Exceptions (documented):**

1. **44px minimum tappable target** — all `<button>` and tab targets must satisfy `min-height: 44px` per WCAG 2.5.5 / Apple HIG. The mobile bottom-tab bar uses `padding: 10px 0` over a 24px emoji to hit ~44px while keeping the 64px bar height.
2. **`env(safe-area-inset-bottom)` additive padding** on the bottom-tab bar and any `<main>` content — required for iOS PWA home-screen install (notched devices). Computed as `max(env(safe-area-inset-bottom, 12px), 12px)`. Not a fixed token.
3. **`var(--radius)` = 16px (Lavender) / 4px (Midnight)** corner radius — theme-dependent, not part of the spacing scale. Carried forward from v1.

## Typography

Exactly 4 roles, exactly 2 weights. Line heights are unitless ratios.

| Role | Size | Weight | Line Height | Used For |
|------|------|--------|-------------|----------|
| Body | 16px | 500 (medium) | 1.5 | Default text — paragraphs, nav labels, form fields, button labels, offline banner text. (v1 used 14–15px globally; bumping to 16px for the rewrite improves mobile readability and matches modern PWA defaults.) |
| Label | 14px | 500 (medium) | 1.4 | Small UI text — subtitle on Login ("Sign in to access your family's dashboard"), Access Denied email display, error pills, theme toggle label. |
| Heading | 20px | 700 (bold) | 1.2 | Family Creation Wizard step titles, placeholder page H1s ("Chores", "Calendar", etc.), Access Denied heading. |
| Display | 28px | 700 (bold) | 1.2 | Login screen "Family Plan" title only. The single display-weight moment in Phase 1. |

**Heading font override (Midnight theme only):** `<h1>–<h4>` switch to `'Noto Serif', Georgia, serif` with `letter-spacing: -0.02em`, per v1 `[data-theme="midnight"]` rules ported verbatim.

**Disallowed in Phase 1:** font sizes other than 14 / 16 / 20 / 28; weights other than 500 and 700; italic; uppercase transforms.

## Color

Lavender (light) is the default. Midnight (dark) is the alternate. Both palettes come from v1 `index.css` and are ported verbatim. The 60/30/10 split below describes screen real estate, not raw pixel counts.

### Lavender theme (default for light OS)

| Role | Value | Variable | Usage |
|------|-------|----------|-------|
| Dominant (60%) | `#fdf6ff` | `--bg` | Page background — app shell, route content area, Login wrapper background |
| Secondary (30%) | `#ffffff` | `--card-bg` | Card surfaces — Login card, Wizard panel, top nav bar, mobile bottom-tab bar, offline banner background |
| Accent (10%) | `#c9a8e0` (mid) / `#9b7bbf` (dark) | `--lavender` / `--lavender-dark` | See "Accent reserved for" list below |
| Destructive | `#d4607a` (text) on `#fad4e0` (bg) | `--pink-dark` / `--pink-light` | Error pill on Login form, Access Denied messaging, error-boundary fallback heading. Sign-out is styled in this pair on hover. |
| Text | `#3d2c4e` body / `#8a7898` muted | `--text` / `--text-muted` | All non-accent text |
| Border | `#e8d5f5` | `--border` | Nav border-bottom, card outline, Google button border, divider lines |

### Midnight theme (default for dark OS)

| Role | Value | Variable | Usage |
|------|-------|----------|-------|
| Dominant (60%) | `#0c0b0f` | `--bg` | Page background |
| Secondary (30%) | `#16141f` | `--card-bg` | Cards, frosted nav (`rgba(22,20,31,0.82)` + `backdrop-filter: blur(24px)`) |
| Accent (10%) | `#e9c176` (warm gold) | `--lavender` (re-mapped per v1 convention — the variable name stays `--lavender` across both themes; only the hex changes) | Same reserved-for list as Lavender theme |
| Destructive | `#f9a8d4` (text) on `#1e0815` (bg) | `--pink-dark` / `--pink-light` | Same surfaces as Lavender |
| Text | `#e5e2e1` body / `#8b8680` muted | `--text` / `--text-muted` | All non-accent text. Warm off-white, never pure white (v1 contract). |
| Border | `rgba(233, 193, 118, 0.12)` | `--border` | Ghost-gold hairlines |

### Accent reserved for (both themes)

The accent color (`--lavender` / `--lavender-dark`) appears in **only** these Phase 1 surfaces:

1. **Active nav tab** — currently-selected route in desktop top nav (pill background) and mobile bottom-tab bar (light tint + 2px upward translate).
2. **Logo word-mark** in the top nav ("Family Plan") — uses `--lavender-dark`.
3. **Primary CTA "Sign in with Google" button on hover/focus** — border switches to `--lavender` and gains `--shadow`.
4. **Primary CTA "Create my family" button (Family Creation Wizard submit)** — solid `--lavender` background with white text.
5. **Form input focus ring** on the Wizard's family-name input — outline color = `--lavender`.
6. **Selected theme chip** in the inline theme toggle (Lavender / Midnight pair).

Accent is **never** applied to: body copy, placeholder route content, nav tab hover state (which uses `--lavender-light` tint instead), the offline banner, the error boundary, or destructive surfaces.

### Theme selection rules (locked in CONTEXT.md D-15)

- **First-ever load:** detect `window.matchMedia('(prefers-color-scheme: dark)').matches` → Midnight; else → Lavender.
- **Subsequent loads:** read `family_settings.theme` from Supabase and apply via `<html data-theme="midnight">` or default `<html>` (no attribute = Lavender).
- **User override:** theme toggle in nav writes to `family_settings.theme` (family-wide, not per-member).

## Copywriting Contract

Every user-facing string in Phase 1 is listed here. Executor copies verbatim.

| Element | Copy |
|---------|------|
| Primary CTA — Login screen | **Sign in with Google** (loading state: **Redirecting…**) |
| Login subtitle | Sign in to access your family's dashboard |
| Login title | Family Plan |
| Access Denied heading | This email isn't on the family list |
| Access Denied body | `{user.email}` doesn't have access to this Family Plan. Ask a parent to add you, then sign in again. |
| Access Denied CTA | **Sign out** |
| Primary CTA — Family Creation Wizard step 1 | **Create my family** |
| Wizard step 1 heading | Name your family |
| Wizard step 1 label | What should we call your family? |
| Wizard step 1 placeholder | The Mader Family |
| Wizard step 1 emoji label | Pick a family emoji |
| Wizard helper text | This is just for you — you can change it later in Settings. |
| Wizard success transition copy (briefly visible on submit) | Setting up your family… |
| Placeholder page H1 (each tab) | Dashboard / Chores / Calendar / Meals / Groceries / Notes |
| Placeholder page body | Coming soon — this section ships in Phase {N}. (Substitute the phase number per ROADMAP.md: Chores=3, Calendar=4, Meals/Groceries/Notes=5, Dashboard stays "Your family at a glance — coming soon.") |
| Offline banner | **Offline** — changes will sync when reconnected |
| Reconnected toast (auto-dismiss 3s) | Back online — syncing your changes |
| Error boundary heading | Something went wrong on this page |
| Error boundary body | We've logged the error. Try reloading, or head back to the dashboard. |
| Error boundary primary CTA | **Reload this page** |
| Error boundary secondary CTA | Back to Dashboard |
| Sign-out button (top nav, desktop + mobile dropdown) | **Sign out** (no confirmation — matches v1 behaviour; sign-out is reversible) |
| Theme toggle labels | Lavender / Midnight (chip pair, currently-selected chip uses accent fill) |

### Destructive actions in Phase 1

| Action | Confirmation approach |
|--------|----------------------|
| Sign out | None — single-tap, reversible by signing back in (matches v1). |
| Access-Denied auto-sign-out | None — fires automatically before Access Denied screen renders, no user-initiated destructive moment. |

There are **no other destructive surfaces** in Phase 1. Member deletion, chore delete, etc. land in Phase 2+ and will declare their own confirmation contracts.

### Voice and tone (applies to every string above)

- **Second person, present tense.** "Sign in to access your family's dashboard" not "Users may sign in here."
- **No exclamation points** anywhere in Phase 1 copy.
- **No emoji in headings or body text** — emoji are reserved for nav tabs and the family avatar picker.
- **Verb-first CTAs.** "Sign in with Google," "Create my family," "Reload this page." Never "Submit," "OK," or "Continue" alone.
- **Surface the email** in Access Denied — the user needs to know which Google account hit the gate so they can pick the right one on retry.

## Interaction Contracts (Phase-1 specific)

These are not in the standard template but are required for the executor and ui-checker to validate behaviour:

| Surface | Interaction |
|---------|-------------|
| Login button | Disabled during `loading`; switches label to "Redirecting…"; opacity 0.6, cursor `not-allowed`. |
| Error pill (Login) | Renders only when `error` is non-null; pink-light background; pink-dark text; full width of card; 8px vertical padding. |
| Top nav (desktop ≥769px) | Sticky `top: 0`, z-index 200, `--card-bg` background, 2px `--border` bottom border. Family name on the left, tab pills in the centre, theme toggle and Sign out on the right. |
| Bottom tab bar (mobile ≤768px) | Fixed `bottom: 0`, z-index 300, full-width, `--card-bg` background, 1px top border, `padding-bottom: max(env(safe-area-inset-bottom, 12px), 12px)`. Six tabs: Dashboard 🏠, Chores ✅, Calendar 📅, Meals 🍽️, Groceries 🛒, Notes 📝. **Note:** v1 cut the 6th tab on mobile bottom-bar (only showed 5); Phase 1 v2 shows all 6 because Notes is now reachable from anywhere. |
| Active tab indicator (mobile) | `--lavender-light` background tint + 2px upward `translateY`. |
| Active tab indicator (desktop) | Solid `--lavender` background, white text, `0 2px 8px rgba(180, 140, 220, 0.3)` shadow. |
| Offline banner | Position: `fixed; top: 0; left: 0; right: 0;` z-index 250 (sits above nav). Height 36px on desktop, 32px on mobile. Slides down from top with a 200ms ease-out transform. Stays visible the entire time `navigator.onLine === false` OR `queryClient` has paused mutations. |
| Reconnected toast | Position: `fixed; top: 12px; right: 12px;` (desktop) or full-width banner below nav (mobile). Auto-dismisses after 3 seconds via `setTimeout`. |
| Error boundary fallback | Centred card, 24px padding, white card-bg, destructive-pink heading, two buttons stacked on mobile, side-by-side on desktop. Reload uses `window.location.reload()`. |
| Theme toggle | Chip pair labelled "Lavender" and "Midnight"; selected chip has accent fill; tap swaps `<html data-theme>` immediately AND writes to `family_settings.theme` in the background. Optimistic — no spinner. |
| Family Creation Wizard | Single-screen for Phase 1 (no multi-step). Card layout mirroring the Login card (same max-width 380px, same `--shadow`, same `--radius`). Submit triggers optimistic `families` INSERT; on success, route transitions to `/dashboard`. |
