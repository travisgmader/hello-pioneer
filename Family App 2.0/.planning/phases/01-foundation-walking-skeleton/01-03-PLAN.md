<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> Supabase Auth (Google OAuth) | OAuth callback returns an authorization code; PKCE prevents code interception attacks. |
| browser -> allowed_emails table | Client queries the table directly through PostgREST under RLS. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | OAuth code interception | mitigate | flowType pkce in src/data/supabase.ts requires the code_verifier to be present on the same client that initiated the flow. Verified by RESEARCH.md Pitfall 9 and the Supabase Auth Google OAuth docs. |
| T-03-02 | Tampering | Open redirect via redirectTo | mitigate | signInWithGoogle hardcodes window.location.origin + trailing slash. Supabase rejects redirectTo values not in the project's allowed redirect URLs list (configured via human checkpoint Task 2.2). |
| T-03-03 | Information Disclosure | Allowlist enumeration | mitigate | RLS SELECT policy on allowed_emails restricts each user to lower(email) = lower(jwt -> email). Even an authenticated attacker can only see their own row, never the whole list. |
| T-03-04 | Tampering | Session lifetime / token theft via XSS | accept (mitigated by stack) | Supabase persistSession writes to localStorage. Phase 1 has zero user-generated HTML rendering, so XSS surface is zero. Phase 5 (Notes) will be the first time we render user-supplied content and must use React's default escaping. |
| T-03-05 | Spoofing | Allowlist case mismatch (TRAVIS@... vs travis@...) | mitigate | isAllowedEmail lowercases + trims before querying. The allowed_emails bootstrap rows are inserted lowercased. The RLS SELECT policy also lowercases both sides. Triple-defense. |
| T-03-06 | Denial of Service | OAuth retry loop on transient Supabase error | accept | The Login button disables during loading and re-enables on error. A user can manually retry. No client-side rate limiting in Phase 1; Supabase backend rate-limits OAuth attempts. |
| T-03-SC | Tampering | npm packages added in this plan | mitigate | This plan adds no new packages. Plan 01 already cleared the package legitimacy checkpoint. |
</threat_model>
