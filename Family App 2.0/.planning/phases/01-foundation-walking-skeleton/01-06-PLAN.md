<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> Vercel CDN | Static assets served from edge; HTTPS enforced. |
| browser -> service worker | SW runs in browser sandbox, intercepts navigations only via precache. |
| Vercel build -> npm packages | Build runs `npm install` against npm registry. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | Service worker caches authenticated Supabase responses | mitigate | src/sw.ts is precache-only — no registerRoute, no fetch handlers, no runtime caching. Cross-origin fetches to Supabase are not intercepted (Pitfall 10). |
| T-06-02 | Tampering | Stale SW serves old build after deploy | mitigate | vercel.json sets /sw.js Cache-Control to max-age=0 must-revalidate. registerType: 'autoUpdate'. src/sw.ts calls cleanupOutdatedCaches and self.clients.claim. Pitfall 4 mitigated. |
| T-06-03 | Tampering | Service worker scope escalation | mitigate | Service-Worker-Allowed: / header is set; scope is explicit at /. Workers cannot escape their declared scope. |
| T-06-04 | Spoofing | Open redirect from Vercel preview URL | accept | The application uses window.location.origin only; Supabase rejects redirectTo values not in the configured allow-list (managed via Plan 03 user_setup). |
| T-06-05 | Information Disclosure | Source maps leak in production bundle | mitigate | Vite default for production build excludes source maps. If sourceMaps is enabled in Phase 6 for error tracking, it must be a separate decision. |
| T-06-06 | Denial of Service | Stuck SW blocking all users post-deploy | mitigate | registerType: 'autoUpdate' + cleanupOutdatedCaches + clients.claim. Worst case: a user can DevTools -> Application -> Service Workers -> Unregister; document this in a runbook. |
| T-06-SC | Tampering | npm packages added in this plan | mitigate | vite-plugin-pwa is the official Vite PWA plugin (vite-pwa org); workbox-precaching and workbox-routing are Google's official Workbox modules. Verified in RESEARCH.md §Package Legitimacy Audit. Re-confirm during npm install. |
</threat_model>
